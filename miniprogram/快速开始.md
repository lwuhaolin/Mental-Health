# 快速开始指南

欢迎使用心理健康助手小程序！本指南将帮助你快速启动和运行这个项目。

## 📋 前置要求

1. **微信开发者工具**
   - 下载地址: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 版本要求: 最新稳定版

2. **后端服务**
   - 确保现有的Spring Boot后端正在运行
   - 默认地址: http://localhost:8080

## 作品安装说明

本作品采用“后端 + 前端 + 小程序端”三部分协同运行，建议按以下顺序安装和启动：

### 1. 安装后端

1. 使用 IntelliJ IDEA 或其他 Java IDE 打开项目根目录 `Psychological counseling`。
2. 确认已安装 **JDK 17** 和 **Maven**。
3. 配置数据库连接信息，检查 `src/main/resources/application.properties` 中的数据库地址、用户名和密码是否正确。
4. 导入数据库文件 `xingyu_hanghai_backup_utf8.sql` 到 MySQL。
5. 启动 Spring Boot 后端服务，默认访问地址为 `http://localhost:8080`。

### 2. 安装前端

1. 进入 `frontend` 目录。
2. 安装依赖：`npm install`。
3. 检查 `src/utils/request.js` 中的接口地址是否与后端一致。
4. 启动开发环境：`npm run dev`。
5. 打包生产版本：`npm run build`。

### 3. 安装小程序端

1. 使用微信开发者工具导入 `miniprogram` 目录。
2. 在 `miniprogram/app.js` 中配置后端接口地址。
3. 关闭开发者工具中的域名校验限制。
4. 点击编译，确认页面能够正常加载。

### 4. 安装完成标准

- 后端服务可正常启动并连接数据库。
- 前端页面可正常访问并请求接口。
- 微信小程序可正常打开首页、AI疗愈、漂流瓶和个人中心页面。

## 设计思路

本作品以“心理健康陪伴与自我认知”为核心，围绕“测评、疏导、咨询、记录”四条主线展开设计。

1. **以用户体验为中心**
   - 通过清晰的首页入口，将 AI 疗愈、心理测评、专家咨询、漂流瓶等功能模块进行聚合。
   - 采用统一的视觉风格和简洁的交互流程，降低用户的使用门槛。

2. **多端协同设计**
   - 后端负责用户、测评、咨询、漂流瓶等核心业务数据管理。
   - 前端用于后台管理和网页端操作展示。
   - 小程序端用于移动场景下的快速访问，提升随时随地的使用便捷性。

3. **功能模块化设计**
   - 将系统拆分为登录注册、AI疗愈、心理测评、专家咨询、漂流瓶、个人中心等独立模块。
   - 每个模块相对独立，便于后续扩展、维护和功能替换。

4. **数据闭环设计**
   - 用户从进入系统、参与测评、获取结果、发起咨询到查看历史记录，形成完整的服务链路。
   - 通过历史记录与个人中心统计，增强用户的持续使用意愿。

## 设计重点与难点

### 设计重点

1. **心理场景的产品表达**
   - 需要在功能完整的基础上体现温和、陪伴、低压的产品氛围。
   - 页面文案、颜色和交互都尽量避免强刺激感，强调舒缓和安全感。

2. **测评流程的完整性**
   - 心理测评需要包含题目加载、答题过程、结果生成和历史记录保存等关键环节。
   - 结果页面不仅展示分数，还要提供可理解的反馈说明。

3. **AI互动的实时性**
   - AI 疗愈对话需要保证响应及时，减少等待焦虑。
   - 同时要兼顾异常兜底，避免接口失败影响整体体验。

4. **匿名社交的可控性**
   - 漂流瓶功能既要支持匿名表达，也要兼顾内容管理与互动秩序。
   - 通过点赞、回复等轻互动方式提升参与感。

### 设计难点

1. **前后端与小程序接口统一**
   - 不同端的请求格式、返回字段和页面结构需要保持一致，否则容易出现数据对接问题。
   - 特别是在登录态、token 传递和结果数据展示上，需要统一规范。

2. **测评结果与历史记录处理**
   - 测评过程中会涉及题目状态、提交时间、结果计算和历史存储。
   - 如何保证结果展示准确、历史记录可追溯，是实现中的重点。

3. **AI接口稳定性**
   - 外部 AI 服务可能出现延迟、限流或失败，需要设计降级方案。
   - 在保证功能可用的前提下，还要处理好错误提示和用户引导。

4. **移动端适配与交互一致性**
   - 小程序需要适配不同尺寸设备，保证页面布局不乱、操作不拥挤。
   - 同时还要保持与前端网页端相近的视觉风格和功能逻辑。

5. **隐私与安全控制**
   - 心理健康类应用涉及用户敏感信息，必须重视数据保护与访问控制。
   - 对用户信息、测评记录和咨询内容需要做好权限与展示边界管理。

## 🚀 启动步骤

### 第一步：配置后端地址

打开 `miniprogram/app.js`，修改 `baseUrl`：

```javascript
App({
  globalData: {
    isLogin: false,
    userInfo: null,
    baseUrl: "http://localhost:8080/api", // 修改为你的后端地址
  },
});
```

**重要提示**：

- 本地开发使用 `http://localhost:8080/api`
- 真机调试需要使用内网IP，如 `http://192.168.1.100:8080/api`
- 生产环境需要使用HTTPS域名

### 第二步：导入项目

1. 打开微信开发者工具
2. 点击 "+" 或 "导入项目"
3. 选择项目目录：`miniprogram` 文件夹
4. 填写以下信息：
   - **项目名称**: 心理健康助手
   - **AppID**: 选择 "测试号" 或填写你的AppID
   - **开发模式**: 小程序

5. 点击 "导入"

### 第三步：开发者工具设置

在开发阶段，需要关闭一些限制：

1. 点击右上角 "详情"
2. 在 "本地设置" 中勾选：
   - ✅ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书
   - ✅ 启用调试模式
   - ✅ 不校验安全域名

### 第四步：编译运行

1. 点击工具栏的 "编译" 按钮
2. 小程序将在模拟器中运行
3. 你应该能看到登录页面

### 第五步：测试登录

使用后端已有的测试账号登录：

```
用户名: test（或你在后端创建的用户）
密码: 你设置的密码
```

**如果没有测试账号**，点击 "注册" 创建一个新账号。

## 🔧 常见问题排查

### 问题1: 网络请求失败

**症状**: 登录时提示"网络连接失败"

**解决方案**:

1. 确认后端服务正在运行
2. 检查 `baseUrl` 配置是否正确
3. 确认已关闭 "校验合法域名" 选项
4. 查看控制台错误信息

### 问题2: 登录后立即退出

**症状**: 登录成功后又跳回登录页

**解决方案**:

1. 检查后端返回的数据格式
2. 确认token字段名称正确
3. 查看 `utils/request.js` 中的响应处理逻辑

### 问题3: 页面空白

**症状**: 某些页面显示空白

**解决方案**:

1. 打开调试器查看错误信息
2. 检查页面路径配置
3. 确认 `app.json` 中的页面路径正确

### 问题4: 真机调试无法访问

**症状**: 真机上提示网络错误

**解决方案**:

1. 将 `baseUrl` 改为电脑的内网IP地址
2. 确保手机和电脑在同一局域网
3. 检查防火墙设置

获取内网IP:

- Windows: 打开cmd，输入 `ipconfig`
- Mac/Linux: 打开终端，输入 `ifconfig`

## 开源代码与组件使用情况说明

本项目在开发过程中主要使用了以下开源框架、组件与工具，均用于提升开发效率、界面体验和系统稳定性：

### 后端技术与组件

- **Spring Boot**：用于构建后端服务与接口。
- **MyBatis Plus**：用于简化数据库访问与持久层开发。
- **MySQL Connector/J**：用于连接 MySQL 数据库。
- **Hutool**：用于常用工具类处理。
- **Lombok**：用于减少样板代码。
- **Swagger Annotations**：用于接口文档注解支持。

### 前端技术与组件

- **Vue 3**：用于构建前端单页应用。
- **Vite**：用于前端开发与打包构建。
- **Element Plus**：用于提供桌面端 UI 组件。
- **Vue Router**：用于页面路由管理。
- **Pinia**：用于状态管理。
- **Axios**：用于 HTTP 请求封装。
- **SCSS**：用于样式编写与维护。

### 小程序端技术与组件

- **微信小程序原生框架**：用于实现移动端小程序页面与交互。
- **自定义组件**：项目中包含头像、页面布局等自定义组件，用于增强复用性与一致性。

### 使用说明

- 本项目所使用的开源组件均遵循各自的开源许可证。
- 项目功能开发以二次整合与业务实现为主，未对外发布第三方商业授权内容。
- 若在后续部署或开源发布时涉及具体许可证要求，应结合实际依赖版本进一步核对。
